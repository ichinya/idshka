# Pint drift blocked verified Auth Socialite work

**Date:** 2026-04-23 14:43
**Files:** tests/Feature/AuthSocialiteFlowTest.php; app/Domain/Sites/*
**Severity:** low

## Problem

Functional verification for `02-user-auth-socialite` passed, but the final verification gate failed because `php vendor/bin/pint --test` reported formatting drift. The primary blocking file was `tests/Feature/AuthSocialiteFlowTest.php`; repository-wide Pint also found existing drift in `app/Domain/Sites/*`.

## Root Cause

The feature test and site-registry files were not run through Laravel Pint before verification. This left code with style that worked functionally but did not match the project formatter, including spacing around unary/not operators, `new` expression parentheses, empty constructor formatting, ordered imports, and Carbon PHPDoc imports.

## Solution

Ran Pint directly on the failing auth test and the authorized `app/Domain/Sites` drift:

```bash
php vendor/bin/pint tests/Feature/AuthSocialiteFlowTest.php app/Domain/Sites
```

This kept the fix mechanical and avoided behavior changes. Verification then passed with repository-wide Pint, targeted auth tests, site-registry tests, full Laravel tests, frontend build, syntax checks, route/event checks, and diff whitespace checks.

## Prevention

- Run `php vendor/bin/pint --test` before `$aif-verify`, not only targeted tests.
- If a plan introduces a new feature test, format it immediately after creation.
- When the working tree contains dirty files from another plan, either exclude them intentionally from plan verification or explicitly clean formatter-only drift if the user authorizes it.
