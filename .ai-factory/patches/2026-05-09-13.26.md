# Compose runtime reached Laravel before database migrations

**Date:** 2026-05-09 13:26
**Files:** `infra/docker/php/entrypoint.sh`, `infra/docker/php/Dockerfile`, `compose.yml`, `README.md`, `tests/Unit/RuntimeHardeningTest.php`
**Severity:** high

## Problem

The local Compose runtime could serve `/portal` before Laravel migrations had been applied. PostgreSQL logged `relation "users" does not exist`, Laravel failed in the session guard user lookup, and nginx could surface a FastCGI 504 while waiting on the failing PHP request. PHP-FPM also printed noisy notices that pool `user` and `group` directives were ignored because FPM was already running as a non-root user.

## Root Cause

Compose only proved that PostgreSQL accepted connections and that PHP existed. It did not prove that the Laravel schema had been created before nginx exposed the app. The entrypoint expected operators to run migrations manually after startup, which leaves a real window where browser traffic can hit an empty schema. The privilege drop used shell-string execution and kept default FPM pool user/group directives that are irrelevant in a rootless FPM process.

## Solution

The entrypoint now always runs `php artisan migrate --force` before PHP-FPM starts. The app healthcheck requires `php artisan migrate:status`, has enough timeout/start period for Windows bind-mount cold starts, and nginx waits for the app to become healthy instead of merely started. The PHP image installs `su-exec`, uses argument-safe privilege dropping, and comments default FPM pool user/group directives so rootless startup logs stay clean. README now documents that local Compose applies pending migrations before PHP-FPM starts.

## Prevention

- Runtime healthchecks should validate the application dependency that caused the incident, not just process availability.
- Local Compose should apply pending migrations unconditionally before exposing nginx; an opt-out flag reintroduces the empty-schema failure mode.
- Healthchecks that run framework CLIs need timeout/start-period budgets that cover cold application startup on bind-mounted development workspaces.
- When running PHP-FPM as a non-root process, remove or override pool directives that only apply to a root master process.
- Keep a focused infrastructure test that asserts Compose waits on a migrated app before exposing public nginx.
