# Hardened operational probes and local runtime defaults

**Date:** 2026-04-20 11:42
**Files:** `.env.example`, `README.md`, `app/Http/Controllers/Api/HealthCheckController.php`, `bootstrap/app.php`, `compose.yml`, `config/database.php`, `infra/docker/php/entrypoint.sh`, `infra/docker/php/fpm-non-root.conf`, `routes/probes.php`, `tests/Feature/FoundationSmokeTest.php`
**Severity:** medium

## Problem

The public `/up` endpoint still used Laravel's built-in HTML maintenance view, which loaded third-party assets and skipped the JSON probe contract plus CSP policy used by the rest of the foundation probes. At the same time, the default Docker Compose profile published PostgreSQL and Redis to `0.0.0.0`, Redis accepted unauthenticated access, and the app container still started `php-fpm` with a root master process.

## Root Cause

The first hardening pass only covered the custom probe routes already owned by the application, leaving Laravel's framework-provided `/up` route in place. Local runtime defaults also favored convenience over least privilege: data services were published directly for host debugging, Redis had no enforced password path, and the container entrypoint still launched `php-fpm` as root because no privilege-drop flow had been wired into the foundation image/runtime config.

## Solution

The framework `/up` route was removed and replaced with the stateless JSON `HealthCheckController` route in `routes/probes.php`, so all public probes now share the same hardened response path and `[FIX:probe-surface]` logging. Compose was tightened to keep PostgreSQL and Redis on the internal Docker network only, Redis now starts with `--requirepass` and falls back to the local database password when `REDIS_PASSWORD` is unset, and `.env.example` plus `README.md` were updated to document the safer local defaults. The app runtime now drops privileges to `www-data` in `entrypoint.sh`, uses a dedicated php-fpm override for writable logs, and the smoke test suite now covers `/up` as a stateless JSON probe.

## Prevention

- Keep operational probe ownership inside application routes instead of relying on framework demo or maintenance endpoints for public infrastructure checks.
- Treat local Docker exposure as opt-in: publish backing services only when a task explicitly needs host access.
- Make local auth defaults fail closed, even in development profiles, so optional env vars do not silently create unauthenticated services.
- Verify container privilege level in runtime checks, not just Dockerfile intent, whenever entrypoints or process managers change.
