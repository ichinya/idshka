# Stateless operational probes and header hardening

**Date:** 2026-04-19 22:58
**Files:** `.env.example`, `app/Http/Controllers/Api/HealthCheckController.php`, `app/Http/Controllers/Api/ReadinessCheckController.php`, `app/Http/Middleware/SetSecurityHeaders.php`, `bootstrap/app.php`, `infra/docker/nginx/default.conf`, `infra/openresty/demo-resource/nginx.conf`, `routes/probes.php`, `routes/web.php`, `tests/Feature/FoundationSmokeTest.php`
**Severity:** medium

## Problem

Foundation routes `/`, `/health` and `/ready` were mounted on the `web` stack and issued `XSRF-TOKEN` plus session cookies to anonymous callers. Public responses also lacked hardening headers, exposed `X-Powered-By`, and the readiness payload leaked backend-specific details such as connection names and Redis ping output.

## Root Cause

The initial foundation implementation optimized for getting Laravel runtime online quickly and reused the default `web` routing surface for operational endpoints. That pulled in session/CSRF behavior by default. Security hardening was also split across future roadmap phases, so no dedicated response-header middleware or proxy-level header policy was introduced in the foundation slice. Finally, readiness was written for operator convenience rather than as a public-safe probe, so it returned internal dependency metadata directly.

## Solution

Operational probes were moved into a dedicated stateless route group (`routes/probes.php`) mounted with `api` middleware from `bootstrap/app.php`. A new global middleware `SetSecurityHeaders` now sets baseline hardening headers and JSON-only CSP, while nginx hides `X-Powered-By` and OpenResty emits equivalent headers for gateway responses. Health and readiness controllers now add `[FIX:probe-surface]` logs, return no-store responses, and expose only coarse component status in the readiness payload. Regression coverage was expanded to assert no `Set-Cookie` headers on the probe endpoints and to verify the new security headers plus sanitized readiness JSON.

## Prevention

- Keep operational probes and public machine endpoints off the `web` stack unless sessions are explicitly required.
- Treat response headers as part of the foundation contract for public-facing identity infrastructure, not as later hardening work.
- When adding health/readiness endpoints, return only the minimum public-safe state and keep dependency detail in internal logs or authenticated ops surfaces.
- Add regression tests for cookies and security headers whenever route middleware or proxy config changes.
