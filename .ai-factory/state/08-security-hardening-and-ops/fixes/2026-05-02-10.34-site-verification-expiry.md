# Site Verification Expiry Retry

## Symptom

`chat.ie0.ru` served the requested `.well-known/idshka-site-verification.txt` token, but verification returned `Verification checked: expired`.

## Root Cause

The verification challenge had a 30 minute TTL and expired before the user checked it. The verifier correctly refused to accept the expired token, but the portal continued showing the old token and there was no retry path for the same owned domain.

## Fix

- Expired unverified challenges are marked `expired`.
- A fresh shared DNS/file token is issued for retry.
- The portal reports that new verification instructions were generated.
- The portal displays the newest verification rows instead of the original expired rows.
