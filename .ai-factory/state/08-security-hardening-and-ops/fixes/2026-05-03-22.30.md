# Fix Trace: Gateway Smoke TTL Resolution

Date: 2026-05-03 22:30:42 +05:00
Change-id: `08-security-hardening-and-ops`
Mode: OpenSpec-native mode

## Selected Finding

`/aif-verify 08-security-hardening-and-ops` reported a blocking gateway smoke failure:

- `stale key after ttl expiry` expected HTTP 401 but received HTTP 200.
- The failing run set the Compose gateway TTL to `GATEWAY_JWKS_CACHE_SECONDS=30`.

## Root Cause

The gateway itself expires retired-key cache entries correctly when the smoke script waits for the actual gateway TTL. The failing path happened because the Compose process received `GATEWAY_JWKS_CACHE_SECONDS=30`, but the shell running `infra/openresty/demo-resource/smoke.sh` could fall back to its default `20` second wait. That made the "after ttl expiry" assertion run before the gateway's real 30-second cache entry expired.

## Fix

- Updated `infra/openresty/demo-resource/smoke.sh` to resolve `JWKS_CACHE_SECONDS` from:
  1. explicit `JWKS_CACHE_SECONDS`,
  2. shell `GATEWAY_JWKS_CACHE_SECONDS`,
  3. running `demo-resource-gateway` container environment,
  4. default `20`.
- Added validation for non-empty positive numeric TTL values.
- Added non-secret diagnostic logging with `[FIX:gateway-smoke-cache-ttl]`.
- Added regression assertions in `tests/Unit/CiWorkflowSecurityTest.php`.

## Evidence

- RED: `php artisan test --without-tty tests/Unit/CiWorkflowSecurityTest.php` failed because the smoke script did not contain runtime TTL resolution.
- GREEN: `php artisan test --without-tty tests/Unit/CiWorkflowSecurityTest.php tests/Unit/GatewayLoggingSafetyTest.php` passed: 5 tests, 46 assertions.
- Syntax: `bash -n infra/openresty/demo-resource/smoke.sh` passed.
- Smoke: with `GATEWAY_JWKS_CACHE_SECONDS=30`, `bash infra/openresty/demo-resource/smoke.sh` passed and logged `[FIX:gateway-smoke-cache-ttl] using JWKS cache TTL 30s`.
