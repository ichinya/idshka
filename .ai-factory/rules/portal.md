# Area: portal

- Secrets показывать только один раз после создания.
- В списках показывать только prefix/fingerprint токена или client secret.
- Опасные действия требуют подтверждения: revoke, delete site, rotate secret, unlink provider.
- UI должен явно показывать режим сайта: API-only, Web login или оба.
- Для domain verification показывать готовые DNS/file инструкции.
