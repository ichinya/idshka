# Fix: Auth Socialite Pint formatting

Date: 2026-04-23 14:43
Plan: `02-user-auth-socialite`
Mode: direct fix

## Problem

`$aif-verify` failed after functional auth/Socialite tests passed because the formatter gate still failed.

Blocking output:

- `tests/Feature/AuthSocialiteFlowTest.php` failed Pint with `new_with_parentheses`, `unary_operator_spaces`, and `not_operator_with_successor_space`.
- Repository-wide Pint also reported existing drift in `app/Domain/Sites/*` files from the dirty `03-site-registry-and-modes` work.

## Root Cause

The new auth feature test and existing site-registry files were written before a repository-wide Pint pass. The implementation was correct, but PHP style did not match the configured Laravel Pint rules.

## Fix

Ran Pint on the exact failing auth test plus the authorized `app/Domain/Sites` drift:

```bash
php vendor/bin/pint tests/Feature/AuthSocialiteFlowTest.php app/Domain/Sites
```

This made style-only changes:

- normalized `new` expressions and unary/not operator spacing in `tests/Feature/AuthSocialiteFlowTest.php`;
- collapsed empty constructors/classes where Pint requires it in `app/Domain/Sites`;
- normalized imports and PHPDoc Carbon references in `app/Domain/Sites` models/services.

No runtime logging was added because this fix is formatting-only and should not alter application behavior. Existing runtime/test logging remains unchanged.
