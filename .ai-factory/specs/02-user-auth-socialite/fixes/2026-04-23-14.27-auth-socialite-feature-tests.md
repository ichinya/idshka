Date: 2026-04-23 14:27

## Problem

`/aif-verify` failed the rules gate because public auth and Socialite endpoints existed without targeted feature/smoke coverage.

## Root Cause

The implementation added route/controller/domain behavior for session auth and Socialite, but the test suite still only covered foundation and site-registry flows. The missing tests also hid a runtime bug in Socialite callback profile normalization.

## Fix

- Added `tests/Feature/AuthSocialiteFlowTest.php`.
- Covered register/login/logout session auth.
- Covered mocked Socialite redirect and callback.
- Covered repeated Socialite callback resolving the existing user.
- Covered account link/unlink.
- Covered unsupported provider errors.
- Covered provider token encryption and log-safety expectations.
- Fixed `Str::limit(...)->toString()` usage in Socialite profile/name handling and request id sanitization, because `Str::limit()` returns a string in the current Laravel version.
- Added request-id sanitization regression coverage in `tests/Feature/FoundationSmokeTest.php` for the same helper-return-type bug.
