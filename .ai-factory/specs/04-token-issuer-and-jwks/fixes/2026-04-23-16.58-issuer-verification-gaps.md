# Fix: issuer verification gaps

## Context

Verification found that the issuer feature tests used synthetic 40-character site IDs, while real `SiteIdFactory` IDs are shorter (`site_` + ULID). The FormRequest rule `size:40` rejected real site IDs before the issuer action could run. The same verification pass also found default Laravel validation error bodies, contradictory issuer/JWKS documentation, and mojibake/trailing whitespace in `DESCRIPTION.md`.

## Root Cause

- `IssueUserApiTokenRequest` required `site_id` to be exactly 40 characters.
- `IssuerApiFlowTest::createSite()` used fake random IDs and masked the mismatch.
- FormRequest validation did not override Laravel's default JSON shape.
- Documentation still described issuer/JWKS as planned after the runtime slice was implemented.
- `.ai-factory/DESCRIPTION.md` had been stored as mojibake text and had trailing whitespace.

## Changes

- Changed issuer request validation to accept real site IDs with `max:40` plus the existing `exists:sites,id` guard.
- Normalized validation failures to deterministic JSON:
  - `error`
  - `message`
  - `request_id`
  - `fields`
- Added a warning log for issuer request validation failures without logging raw tokens.
- Updated issuer feature/unit tests to create sites through `SiteIdFactory`.
- Added a regression test for the deterministic validation error shape.
- Updated docs to mark user API token issue/revoke and `GET /oauth/jwks.json` as implemented runtime slices, while keeping gateway validation and OAuth/OIDC provider endpoints as future phases.
- Restored `.ai-factory/DESCRIPTION.md` text encoding and removed trailing whitespace in touched files.

## Verification

- `php artisan test --without-tty --filter=IssuerApiFlowTest` failed before the fix on real site IDs and validation error shape.
- `php artisan test --without-tty --filter=IssuerApiFlowTest` passed after the fix: 5 tests, 52 assertions.
- `composer validate --strict` passed.
- `php vendor\bin\pint --test` passed.
- `php artisan test --without-tty` passed: 43 tests, 324 assertions.
- `git diff --check` passed with only CRLF normalization warnings for markdown files.
- Targeted trailing-whitespace scan returned 0 matches for touched code/docs.
- Contradictory issuer/JWKS text search returned no matches.
- `php artisan route:list --path=api/v1/user` shows token issue/revoke routes.
- `php artisan route:list --path=oauth` shows `GET|HEAD oauth/jwks.json`.

## Notes

One extra route-list sanity check was first run with unsupported `--compact`; Artisan rejected the option. The command was rerun without that option and passed.
